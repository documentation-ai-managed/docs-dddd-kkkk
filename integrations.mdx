## Git Repository Syncing

Keep your Dddd Kkkk documentation in sync with your Git repository. Changes pushed to your repo trigger automatic rebuilds and updates.

<Steps>
  <Step title="Connect Repository" icon="git-branch">
    Navigate to Settings > Integrations > Git Sync. Enter your repository URL, such as `https://github.com/your-org/your-docs-repo.git`.
  </Step>
  <Step title="Authenticate" icon="key">
    Authorize Dddd Kkkk using your Git provider's OAuth flow or a personal access token.
  </Step>
  <Step title="Configure Branch" icon="git-commit">
    Select the branch (e.g., `main`) and set sync frequency to every push or on schedule.
  </Step>
  <Step title="Test Sync" icon="play">
    Trigger a manual sync to verify. Monitor the status in the integrations dashboard.
  </Step>
</Steps>

<Callout kind="tip">
  Use branch protection rules in your Git repo to ensure only reviewed changes sync to production documentation.
</Callout>

## Webhook Configurations

Webhooks notify external services about events like document updates or new versions. Configure them in Settings > Webhooks.

```javascript
// Example webhook payload for a document update
{
  "event": "document.updated",
  "doc_id": "doc_123abc",
  "timestamp": "2024-10-15T12:00:00Z",
  "changes": ["title", "content"]
}
```

<Request tabs="cURL,JavaScript" show-lines="true">
  ```bash
  curl -X POST https://your-webhook-url.com/ddd-kkkk \
    -H "Content-Type: application/json" \
    -d '{
      "event": "document.updated",
      "doc_id": "doc_123abc"
    }'
  ```
  ```javascript
  fetch('https://your-webhook-url.com/ddd-kkkk', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      event: 'document.updated',
      doc_id: 'doc_123abc'
    })
  });
  ```
</Request>

## Third-Party App Connections

Connect Dddd Kkkk to popular services for enhanced workflows.

<Tabs>
  <Tab title="Zapier" icon="zap">
    Create "Zaps" to trigger actions on document events.
    
    <Steps>
      <Step title="Setup Zapier">
        Search for Dddd Kkkk in Zapier and authenticate with your API key (`YOUR_API_KEY`).
      </Step>
      <Step title="Choose Trigger">
        Select events like "New Document" or "Update Published".
      </Step>
    </Steps>
  </Tab>
  <Tab title="Slack" icon="message-circle">
    Post notifications to Slack channels.
    
    ```bash
    # Webhook URL from Slack app settings
    WEBHOOK_URL="https://hooks.slack.com/services/YOUR/SLACK/WEBHOOK"
    ```
  </Tab>
  <Tab title="GitHub Actions" icon="github">
    Automate docs deployment in CI/CD.
    
    <CodeGroup tabs="YAML">
      ```yaml
      name: Deploy Docs
      on: push
      jobs:
        deploy:
          runs-on: ubuntu-latest
          steps:
            - uses: actions/checkout@v4
            - name: Sync to Dddd Kkkk
              run: curl -X POST https://api.example.com/v1/sync \
                -H "Authorization: Bearer YOUR_API_KEY" \
                -d '{"repo": "your-repo"}'
      ```
    </CodeGroup>
  </Tab>
</Tabs>

## API Access for Custom Integrations

Build bespoke integrations using our REST API at `https://api.example.com/v1`.

<ParamField path="docs/:id" param-type="GET" required="true">
  Retrieve a specific document.
</ParamField>

<ParamField header="Authorization" param-type="string" required="true">
  Bearer token: `Authorization: Bearer YOUR_API_KEY`.
</ParamField>

<Response tabs="200,401">
  ```json
  {
    "id": "doc_123abc",
    "title": "Integration Guide",
    "content": "Updated content...",
    "updated_at": "2024-10-15T12:00:00Z"
  }
  ```
  ```json
  {
    "error": "Unauthorized",
    "message": "Invalid API key"
  }
  ```
</Response>

<Expandable title="Advanced API Usage" default-open="false">
  For bulk operations, use the `/docs/batch` endpoint. Rate limits apply: `<1000` requests per hour.
  
  ```javascript
  // Batch update example
  const batchUpdate = async (docs) => {
    const response = await fetch('https://api.example.com/v1/docs/batch', {
      method: 'POST',
      headers: {
        'Authorization': `Bearer ${YOUR_API_KEY}`,
        'Content-Type': 'application/json'
      },
      body: JSON.stringify(docs)
    });
    return response.json();
  };
  ```
</Expandable>

<Callout kind="info">
  Generate your API key in Settings > API Access. Store it securely and rotate regularly.
</Callout>
